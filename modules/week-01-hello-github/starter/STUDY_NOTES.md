## Git Workflow Understanding

**What is the difference between `git add`, `git commit`, and `git push`?**
[Explain each command and when you would use it]

Answer: `git add` takes a modified file in your working directory and places the modified version in a staging area.
You would use `git add` after modifying and saving one or more files.
`git commit` takes everything from the staging area and makes a permanent snapshot of the current state of your
repository that is associated with a unique identifier. You should use `git commit` whenever you reach a stable and
logically complete stage of work that you want to save. Committing frequently acts as a safety point, allowing you
to easily revert to a previous working state just in case something goes wrong.
`git push` uploads local commits from your local repository to a specified remote repository like GitHub. You should
use `git push` when you are ready to upload your local repository to GitHub. This way, you will have a remote backup
of your work and can collaborate with others.

**Why do we create branches?**
[Explain the purpose of creating a student branch]

Answer: Branches are primarily used to isolate development work, allowing developers to work on new features or bug
fixes in parallel without affecting the stability of the main codebase. This prevents broken or unstable code from
reaching the production version of the software.
